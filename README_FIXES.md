# Исправления авторизации и админ-панели

## Что было исправлено:

### 1. Авторизация (LoginPage.vue)
- **Проблема**: В методе `login()` был мёртвый код с обращением к переменной `data` до её объявления
- **Решение**: Удалён нерабочий код, теперь логин работает корректно

### 2. CORS (Program.cs)
- **Проблема**: Использовалась политика "AllowFrontend", которая не была определена
- **Решение**: Изменено на "AllowAll" (разрешает все origin для разработки)

### 3. Админ-панель (AdminController.cs + AdminPage.vue)
**Добавлен новый контроллер администратора:**
- `GET /api/admin/users` - получение всех пользователей
- `PUT /api/admin/users/{id}/role` - изменение роли пользователя
- `DELETE /api/admin/users/{id}` - удаление пользователя
- `GET /api/admin/stats` - статистика системы

**Обновлён frontend админ-панели:**
- Статистика (пользователи, компоненты, сборки, результаты квизов)
- Таблица пользователей с управлением ролями
- Таблица компонентов с CRUD операциями
- Модальные окна для добавления/редактирования компонентов

### 4. Компоненты (ComponentsController.cs)
Добавлены методы для админки:
- `PUT /api/components/{id}` - обновление компонента
- `DELETE /api/components/{id}` - удаление компонента

### 5. AuthController.cs
- Добавлено поле `Id` в ответ метода `/auth/me`

### 6. HomePage.vue
- Добавлено поле `Id` в объект пользователя

## Тестовые учётные данные:
- **Админ**: admin@example.com / admin12345
- **Обычный пользователь**: создаётся через регистрацию

## Запуск:
```bash
# Backend (в терминале 1)
cd backend
dotnet run

# Frontend (в терминале 2)
cd frontend
npm run dev
```

## Примечания:
- Админ-панель доступна только пользователям с ролью "admin"
- Последний администратор не может быть удалён или понижен
- Все эндпоинты админки защищены `[Authorize(Roles = "admin")]`
