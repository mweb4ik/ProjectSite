# PC Architecture Guide

## О проекте
**PC Architecture Guide** — учебное веб-приложение по устройству ПК. Проект совмещает:
- каталог комплектующих,
- интерактивный конфигуратор сборки,
- проверку совместимости,
- образовательные разделы (BIOS, лаборатория, викторина),
- авторизацию и роли пользователей.

## Основные возможности

### 1) Каталог комплектующих
- Просмотр списка компонентов с карточками (цена, характеристики, сокет, энергопотребление, изображение).
- Поиск по названию.
- Фильтрация по категориям: CPU, MB, RAM, GPU, SSD, COOL.
- Пагинация с кнопкой «Загрузить ещё».

### 2) Конфигуратор сборки (Builder)
- Выбор категории и добавление компонента в сборку.
- **Ограничение: по каждой категории можно выбрать только 1 компонент** (например, один CPU, одна RAM и т.д.).
- Кнопка добавления у карточки меняет состояние (в сборке/недоступно), если категория уже занята.
- Расчёт итоговой цены и суммарного энергопотребления.
- Проверка совместимости и сохранение сборки.

### 3) Пользователи и доступ
- Вход/восстановление пароля.
- Роли и защищённые разделы.
- Профиль пользователя и результаты викторины.

-  *Представляет собой викторину с тематическими вопросами по комплектующим компьютера*

       накопители, БП, корпус, охлаждение*
  *Симуляция сборки с прогрессом заполнения — визуализация занятых слотов, свободных разъёмов, прогноза
       кабель-менеджмента*
  *Сравнение «бюджет vs производительность» — график распределения бюджета по компонентам, рекомендации по
       оптимизации цены*
- **8.Раздел с BIOS/UEFI - база прошивок для материнских плат с инструкциями по обновлению, предупреждениями о рисках, историей версий, откатом к предыдущей**;
  Представляет собой: *Интерактивная история версий — таймлайн всех версий BIOS для выбранной платы с датами выхода, изменениями,

  *Симулятор обновления с прогресс-баром — пошаговая визуализация процесса прошивки с предупреждениями на каждом
       этапе*
  *Таблица совместимости CPU ↔ BIOS — динамическая проверка: какая версия нужна для поддержки конкретного
       процессора, можно ли откатиться*

## Модели данных

**User** — `id, username, email, passwordHash, role (Guest/User/Admin), createdAt`

**Component** — `id, name, category (CPU/GPU/RAM/etc.), manufacturer, price, specifications, socket, powerConsumption`

**PCBuild** — `id, userId, name, componentIds[], totalPrice, isCompatible`

**QuizResult** — `id, userId, score, totalQuestions, completedAt`

**QuizQuestion** — `id, question, options[], correctAnswer, difficulty`

**BiosVersion** — `id, motherboardId, version, releaseDate, description, stability, isBeta`

**Motherboard** — `id, name, socket, chipset`

**CpuSupport** — `id, cpuName, biosVersionId, isSupported`

**OverclockProfile** — `id, userId, cpuName, frequency, voltage, temperature, stability, createdAt`

## Локальный запуск

### 1. Подготовка
Установите:
- Node.js (LTS)
- npm
- .NET SDK 8

### 2. Frontend
```bash
cd frontend
npm install
npm run dev
```

### 3. Backend
В отдельном терминале:
```bash
cd backend
dotnet run
```

После запуска:
- Frontend обычно доступен на `http://localhost:5173`
- Backend обычно доступен на `http://localhost:5xxx` / `https://localhost:7xxx` (смотрите вывод `dotnet run`)

## База данных и миграции

В проекте используется EF Core и миграционный подход к изменениям схемы БД.

Базовые команды:
```bash
cd backend
# создание новой миграции
dotnet ef migrations add <MigrationName>

# применение миграций к БД
dotnet ef database update
```

> В репозитории уже есть папка `backend/Migrations`, а база построена вокруг модели миграций.

## API
